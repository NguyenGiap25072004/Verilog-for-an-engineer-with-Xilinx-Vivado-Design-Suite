Q) Is creating a temporary variable a standard coding practice? Instead can we make output y as a wire and assign directly avoiding temporary variable.

let us suppose a,b are inputs, sel is select line input and y is output.

Behavioral Modeling can be done in two ways here.

1) we can write equation for the output using continuous assignment as

assign y = (sel == 1'b1) ? a : b ; using ternary Operator

so what we are doing above is checking whether sel = 1, if it is true we will assign output y to a else we will connect y to b.

2) We can also write the equation of the output using procedural assignment but since it only support reg. we need to declare output as reg if we wish to use Procedural assignment.

reg temp = 1'b0;

always@(*)

begin

if(sel == 1'b1)

temp = a;

else

temp = b;

end

assign y = temp;


Since procedural Modeling only support reg datatype we need to use it to implement Mux.

Hope this clear your doubt.

Wire datatype will be require when we implement 2:1 Mux using structural Modeling style.

y = sel_bar * b + sel * a;

so we can use AND gate , NOT gate and OR gate to implement MUX where we will have undefined nets which can be declared with the wire datatype.
