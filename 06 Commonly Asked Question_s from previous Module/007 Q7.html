Q . What are possible values of == operator does it include x or not other than 0,1 if === are there we can expect z,x as output.

( NV Sai Bhaskar)


Sol.

In Verilog, reg datatype could take 4 different values such as 0,1, X and Z.

1) == operator work perfectly to compare variable with 0 and 1 values but do not work with X and z values.


CASE 1: Number match


module tb();
  
  reg [1:0] a = 2'b11;
  
  
  initial begin
    if(a == 2'b11)
      $display("Number Match");
    else
      $display("Number do no match");
    end
  
endmodule


CASE 2: Number Mismatch

module tb();
  
  reg [1:0] a = 2'b10;
  
  
  initial begin
    if(a == 2'b11)
      $display("Number Match");
    else
      $display("Number do no match");
    end
  
endmodule



CASE 3: When we feed X or Z value it will always show Number Mismatch as == operator are in capable of performing a comparison with X and Z value. Assume due to initialization we set a to 1x and have specified one of the state as 1x still equality operator won't able to compare. Observe the example mentioned below.


module tb();
  
  reg [1:0] a = 2'b1x;  
  
  
  initial begin
    if(a == 2'b11)
      $display("Number Match");
    else if (a == 2'b1x) ///we have feed valid value matching to a (1x) but still it will show can't compare
      $display("One Initialized Value");
    else
      $display("Can't Compare");
    end
  
endmodule



CASE 4: === will perfectly able to compare x and z values.

module tb();

  reg [1:0] a = 2'b1x;  
  
  
  initial begin
    if(a === 2'b11)
      $display("Number Match");
    else if (a === 2'b1x) 
      $display("One Initialized Value");
    else
      $display("Can't Compare");
    end
  
endmodule
